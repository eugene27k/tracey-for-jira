# Tracey Privacy Policy Draft

Effective date: To be set before Marketplace publication.

Tracey is operated by Eugene K. This policy describes how Tracey handles data when installed in a Jira Cloud site.

## Data Processed

Tracey processes Jira data required to render traceability views, including:

- Issue keys, summaries, issue types, statuses, priorities, fix versions, labels, parent issue references, and issue links.
- Values from Tracey custom fields, including Affected Components, User Roles, Permissions, DB Entities, and Use cases.
- Project metadata needed for filtering, such as components, issue types, epics, and versions.
- App configuration and saved view state.

## Data Storage

Tracey stores app configuration and view state in Atlassian Forge app storage.

Tracey does not intentionally store Jira issue content outside Atlassian Forge storage.

## Data Sharing

Tracey does not sell customer data.

Tracey does not send Jira data to third-party analytics or advertising services.

Tracey may link users to external documentation, support, or vendor contact pages when users choose to open those links.

## Data Access

Tracey accesses Jira data through Atlassian Forge APIs using the permissions granted during installation.

Users can only view data that Jira and the installed app permissions allow the app to access.

## Data Retention

Tracey retains app configuration and saved view state while the app remains installed.

When Tracey is uninstalled, Atlassian Forge app storage retention and deletion behavior applies according to Atlassian's platform policies.

## Security

Tracey runs on Atlassian Forge and does not operate a separate external backend for core app processing.

Security issues can be reported to eugenktheba@gmail.com.

## Contact

For privacy questions, contact eugenktheba@gmail.com.
